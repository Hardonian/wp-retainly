# Contributing

Thanks for contributing.

## Workflow

1. Open an issue describing the change (bug, feature, or docs).
2. Create a branch from `main`.
3. Keep PRs focused and small.
4. Include tests or validation notes.
5. Pass CI checks before requesting review.

## Pull Request Checklist

- [ ] Problem statement is clear
- [ ] Change is scoped and documented
- [ ] Security and backward compatibility considered
- [ ] Tests added/updated where relevant

## Standards

- Prefer clarity over cleverness
- Keep behavior observable and debuggable
- Avoid hidden side effects
